# PDF Generation

## Primary approach: Chrome print (recommended)

The HTML is A4-optimised with `@page { size: A4; margin: 16mm 18mm 18mm 18mm }`.
The simplest and highest-fidelity conversion is via Chrome's print dialog.

**Tell the user:**
> "To convert to PDF: open in Chrome → Cmd/Ctrl+P → Save as PDF → enable **Background graphics** → Save."

The **Background graphics** checkbox is essential — without it Chrome strips all background
colors (score bar fills, colored tags, red header bar). The CSS rule
`print-color-adjust: exact !important` enforces this on the browser side, but the
checkbox must also be enabled.

---

## Why background colors may not print

Chrome by default does not print background colors for print performance reasons.
The HTML files include this rule to override that behavior:

```css
@media print {
  * { -webkit-print-color-adjust: exact !important; print-color-adjust: exact !important; }
}
```

This covers: score bar fills, use case color tags, red header bar, evidence block
backgrounds, stat grid backgrounds. If a user reports missing colors in the PDF,
verify this rule is present in the `<style>` block.

---

## Why Google Fonts CDN must NOT be used

When a user opens a local `.html` file via `file://` in Chrome, Chrome blocks all
external network requests (including `fonts.googleapis.com`) for security reasons.
This causes the page to fail to render or render with broken fonts.

All HTML files generated by this skill must use only system fonts:
```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Helvetica, Arial, sans-serif;
/* For monospace (numbers, code): */
font-family: monospace;
```

Never include:
```html
<!-- DO NOT ADD THIS -->
<link href="https://fonts.googleapis.com/css2?family=IBM+Plex+Sans..." rel="stylesheet">
```

## Programmatic approach (optional)

If a PDF file is explicitly requested, use weasyprint:

```bash
pip install weasyprint --break-system-packages
python3 -c "
from weasyprint import HTML
HTML(filename='input.html').write_pdf('output.pdf')
"
```

Note: weasyprint may not render all CSS features perfectly. The Chrome print approach
always produces higher-fidelity output for complex A4 layouts.
